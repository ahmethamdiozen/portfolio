## Overview

A cloud-hosted RAG (retrieval-augmented generation) system built without high-level orchestration frameworks. PDFs are chunked with overlapping windows, embedded via OpenAI, and stored in ChromaDB. Answers stream token-by-token to the frontend and are grounded-checked against retrieved context before source citations are returned.

## Stack

- **FastAPI** — upload, query, and streaming SSE API
- **OpenAI** — `text-embedding-3-small` for embeddings, `gpt-4o-mini` for completions
- **ChromaDB** — persistent vector store with metadata filtering
- **pypdf** — page-aware PDF text extraction
- **Next.js** — drag-and-drop upload, streaming answer display, source citations

## Notes

The RAG pipeline is implemented from scratch — no LangChain. The grounding check calls a secondary LLM to verify the answer is actually supported by the retrieved chunks before exposing source citations, reducing hallucination risk.
